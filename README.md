Sistema de Gestão de Eventos Acadêmicos (SGEA) 
O SGEA é um sistema web desenvolvido com o objetivo de facilitar o gerenciamento de 
eventos acadêmicos, como seminários, palestras, minicursos e semanas acadêmicas.   
A plataforma permite que alunos, professores e organizadores realizem cadastros, inscrições e 
emissão de certificados de forma simples, centralizada e segura. 
Descrição do Projeto 
O projeto foi desenvolvido como parte das atividades avaliativas da disciplina Programação 
para Web e tem como foco o aprendizado prático de boas práticas de programação, 
estruturação de código e integração entre front-end, back-end e banco de dados. 
O SGEA busca automatizar processos acadêmicos que antes eram realizados manualmente, 
otimizando o tempo dos organizadores e garantindo mais autonomia aos alunos e professores. 
Público-alvo - Alunos e professores de instituições de ensino. - Organizadores de eventos acadêmicos. 
Funcionalidades principais - Cadastro e autenticação de usuários. - Criação e edição de eventos acadêmicos. - Inscrição de usuários em eventos disponíveis. - Emissão de certificados digitais. - Gerenciamento de inscrições e presença. - Visualização de eventos disponíveis e destaque na tela inicial. 
Tecnologias Utilizadas -  Python  -  Django Framework  -  SQLite3 - HTML5, CSS3 - Figma - Git e GitHub 
Passo a passo da instalação - Extrair o projeto - Colocar no VSCode e abrir a pasta que contém o projeto - Abrir o terminal e colocar: “python manage.py runserver” 
- Acesse o link usando o “Ctrl + botão esquerdo do mouse” 
Instruções de Uso 
Após instalar e executar o sistema, os usuários podem: 
Alunos e Professores - Criar uma conta ou fazer login. - Consultar os eventos disponíveis na tela inicial. - Realizar inscrições e baixar certificados disponíveis. 
Organizadores - Criar novos eventos e definir informações como datas, local e número de vagas. - Visualizar inscrições e emitir certificados para os participantes. - Editar ou excluir eventos cadastrados. 
Código de conduta - Mantenha um ambiente de colaboração respeitoso. - Evite alterações sem explicação ou que quebrem o funcionamento atual do sistema. - Sempre documente suas mudanças no código e no README.
